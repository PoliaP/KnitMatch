# Техническое задание на проект: «KnitMatch»

# 1. Цель и концепция проекта

**Проблема**: У рукодельниц и вязальщиков накапливаются остатки пряжи различных цветов, составов и толщин. Поиск подходящих схем вязания для конкретных остатков — трудоемкий процесс, требующий ручного сопоставления характеристик пряжи с требованиями схем.

**Цель**: Создание сервиса, который поможет вязальщицам эффективно использовать остатки пряжи.Пользователи добавляют информацию о своей пряже (тип, цвет, количество), система автоматически загружает популярные схемы вязания из крупнейшей базы Ravelry и рекомендует подходящие схемы на основе типа пряжи.

**Решение:** Веб-сервис KnitMatch, который:
1. Анализирует остатки пряжи пользователя (вес, состав, количество)
2. Автоматически загружает актуальную базу схем из крупнейшего внешнего каталога (Ravelry API)
3. Интеллектуально рекомендует модели, которые можно связать из имеющихся материалов
4. Визуализирует статистику и прогресс пользователя

# 2. Роли пользователей

Гость: Может регистрироваться/входить, просматривать демо-страницу

Пользователь: Может добавлять свою пряжу и получать рекомендации схем

# 3. Модели данных

__Модель 1__: UserYarn (Пряжа пользователя)

user (ссылка на User): Владелец

yarn_type (текст): Тип пряжи (тонкая, средняя, толстая)

color (текст): Основной цвет

amount (число): Количество в граммах


__Модель 2__: Pattern (Схема вязания)

ravelry_id (текст): ID из Ravelry

name (текст): Название схемы

yarn_weight (текст): Требуемая толщина пряжи

photo_url (ссылка): Ссылка на фото

source (текст): "ravlery" или "manual"


__Модель 3__: Favorite (Избранное)

user (ссылка на User): Пользователь

pattern (ссылка на Pattern): Схема

added_at (дата-время): Когда добавлено

# 4. Ключевой функционал

-Регистрация и вход пользователей (стандартная Django auth)
-Пользователь добавляет свою пряжу через простую форму (3 поля)
-При первом запуске система автоматически загружает 20 популярных схем из Ravelry API
-На главной странице показываются схемы, подходящие под пряжу пользователя
-Пользователь может добавлять схемы в "Избранное"
-Простая статистика: сколько пряжи каждого типа у пользователя

# 5. Внешние интеграции и аналитика

**Интеграция**: Использование внешнего Ravelry API для получения базы популярных схем вязания. API используется для однократной загрузки 20 самых популярных схем по запросу "knitting" с параметрами сортировки по популярности и ограничением на количество результатов.

**Аналитика**: Использование библиотеки Pandas для анализа данных пользователя: расчет общего количества пряжи по типам, определение наиболее распространенных цветов.
